# Security Policy

## Reporting a Vulnerability

Please report security issues privately.

Preferred path:
- GitHub Security Advisory / private vulnerability reporting for this repository.

If private reporting is not available in your interface, open a regular issue with minimal details
and request a private follow-up channel. Please avoid posting exploit details publicly before
maintainers can triage.

When reporting, include:
- affected component/crate
- affected versions/commit range (if known)
- impact summary
- reproduction steps or proof-of-concept (private)
- suggested mitigation (optional)

## Supported Versions

Tileline is under active pre-alpha development. Security support is best-effort.

| Version line | Security support |
|---|---|
| `master` | Best effort (active development) |
| `v0.5.0` (after release) | Planned best effort patch support |
| `<= v0.4.5.x` | Limited / no guaranteed backports |

## Triage Expectations

Maintainers aim to:
- acknowledge reports quickly
- confirm severity and impact
- provide mitigation guidance
- publish fixes/notes when ready

Response times may vary based on release-critical work.
