# Tileline v0.4.5.1 Release Notes

Release date: 2026-03-22

## Scope

`v0.4.5.1` is a focused 2D sprite-content patch release on top of `v0.4.5`.

This patch adds native `.png` and `.svg` sprite texture source support to the
runtime `.tlsprite` path while preserving the `v0.4.5` project/packaging flow.

## Highlights

- `.tlsprite` sprite entries now support external texture sources:
  - `texture = <path>.png|<path>.svg`
  - `sprite_texture = ...` alias
- Runtime infers deterministic atlas slots for texture-backed sprites when
  `texture_slot` is omitted (external image lane reserved in `[64..127]`).
- Runtime decodes `.png` and rasterizes `.svg` into RGBA atlas tiles at bind time.
- `TlspriteProgram::sprite_texture_bindings()` now exposes unique texture upload
  bindings for TLApp/runtime renderers.
- Added parser/runtime validation coverage for `.png` texture-source inference.

## Included Binary

Primary binary for this release:

- `runtime` crate: `tlapp`

Release artifacts are generated into:

- `dist/v0.4.5.1/<target-triple>/`

Expected files:

- `tlapp` (or `tlapp.exe` on Windows)
- `SHA256SUMS`
- `build-info.txt`
- `tileline-v0.4.5.1-<target-triple>-tlapp.tar.gz`

## Build and Package

```bash
./scripts/release_v0.4.5.1.sh
```

Optional cross-target build:

```bash
TARGET_TRIPLE=aarch64-apple-darwin ./scripts/release_v0.4.5.1.sh
```

Project-root `.pak` build (numbered 5GB shards):

```bash
./scripts/build_tlpfile_pak.sh \
  --project docs/demos/tlapp/tlapp_project.tlpfile \
  --out-dir dist/pak \
  --base-name tlapp-assets \
  --max-shard-gb 5
```

## Run

```bash
./dist/v0.4.5.1/<target-triple>/tlapp \
  --project docs/demos/tlapp/tlapp_project.tlpfile \
  --scene main
```

before broader `v0.5.0` render-stack and independence milestones.
