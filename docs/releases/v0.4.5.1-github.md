# Tileline v0.4.5.1

Date: 2026-03-22

Tileline `v0.4.5.1` is a focused 2D content-path patch release.
It adds runtime `.png/.svg` sprite texture support to `.tlsprite` while
preserving the `v0.4.5` project-root `.tlpfile` + `.pak` workflow.

## Highlights

- `.tlsprite` now supports sprite texture keys:
  - `texture = <path>.png|<path>.svg`
  - `sprite_texture = ...` (alias)
- Runtime infers deterministic image atlas slots in `[64..127]` when `texture_slot` is omitted.
- Runtime decodes `.png` and rasterizes `.svg` to atlas RGBA tiles on bind.
- `TlspriteProgram::sprite_texture_bindings()` exposes unique texture bindings for renderer upload.
- Added parser/runtime test coverage for `.png` inference flow.

## Quick Start

Build release artifact:

```bash
./scripts/release_v0.4.5.1.sh
```

Build project package shards:

```bash
./scripts/build_tlpfile_pak.sh \
  --project docs/demos/tlapp/tlapp_project.tlpfile \
  --out-dir dist/pak \
  --base-name tlapp-assets \
  --max-shard-gb 5
```

Run TLApp from project manifest:

```bash
./dist/v0.4.5.1/<target-triple>/tlapp \
  --project docs/demos/tlapp/tlapp_project.tlpfile \
  --scene main
```
