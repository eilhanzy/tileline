# Tileline v0.4.5.1

Date: 2026-03-27

Tileline `v0.4.5.1` is a patch-stability update on top of `v0.4.5`:
it keeps the 2D content-path improvements (`.png/.svg` sprite textures) and adds
runtime-driven `.tlscript` contact/effect controls with strict validation.

## Highlights

- `.tlsprite` now supports sprite texture keys:
  - `texture = <path>.png|<path>.svg`
  - `sprite_texture = ...` (alias)
- Runtime infers deterministic image atlas slots in `[64..127]` when `texture_slot` is omitted.
- Runtime decodes `.png` and rasterizes `.svg` to atlas RGBA tiles on bind.
- `TlspriteProgram::sprite_texture_bindings()` exposes unique texture bindings for renderer upload.
- Added parser/runtime test coverage for `.png` inference flow.
- Added global contact/touch built-ins in `.tlscript`:
  - `contact_any/contact_pairs/contact_manifolds`
  - aliases: `touch_any/touch_pairs/touch_manifolds`
- Added scripted effect controls in `.tlscript` (no new render pass):
  - `set_render_distance(v)`
  - `set_adaptive_distance(mode)`
  - `set_distance_blur(mode)`
  - `set_msaa(samples)`
- Runtime merge precedence is deterministic:
  - `CLI script overlay > scene script (.tlscript/.tljoint) > current runtime state`

## Quick Start

Build release artifact:

```bash
./scripts/release_v0.4.5.1.sh
```

Build project package shards:

```bash
./scripts/build_tlpfile_pak.sh \
  --project docs/demos/tlapp/tlapp_project.tlpfile \
  --out-dir dist/pak \
  --base-name tlapp-assets \
  --max-shard-gb 5
```

Run TLApp from project manifest:

```bash
./dist/v0.4.5.1/<target-triple>/tlapp \
  --project docs/demos/tlapp/tlapp_project.tlpfile \
  --scene main
```
