# Tileline v0.4.0

Date: 2026-03-18

Tileline `v0.4.0` is a performance release that parallelises every remaining
serial hot path in the ParadoxPE physics engine and introduces an MPS-backed
async physics pipeline that overlaps the physics step with the GPU upload
window, eliminating back-to-back CPU serialisation on the main thread.

## Highlights

All numbers are measured at **10 000 bodies / 2 substeps**, Ryzen 9 7900 (24 threads).

| Subsystem | Before | After | Gain |
|-----------|-------:|------:|-----:|
| Broadphase — parallel AABB sweep (`par_iter_mut`) | 2 457 µs | 769 µs | **−69 %** |
| Narrowphase — reuse pre-allocated `produced_buffer` | 519 µs | 121 µs | **−77 %** |
| Solver — sparse Jacobi clear/flush (~210 slots vs 10 K) | 1 220 µs | 173 µs | **−86 %** |
| Sleep — stride-gate fix + 6 K-body tier + early-skip | 731 µs | 62 µs | **−92 %** |

**MPS async pipeline** — `world.step(dt)` is submitted as a `Critical`-priority
MPS task before `upload_draw_frame`. Physics runs concurrently with the GPU
upload window; `token.wait()` at the next frame start returns near-instantly.

**Bounce showcase raised to 30 000 bodies** — demonstrating sustained multi-core
utilisation at production scale on 24-thread desktop hardware.

## What Changed

- `paradoxpe`: broadphase, narrowphase, solver, sleep — parallelisation and
  allocation/stride fixes across all four subsystems.
- `runtime`: new `PhysicsMpsRunner` + `PhysicsStepToken`; restructured render
  loop to pipeline physics across the GPU upload window.
- `bounce_showcase.tlscript`: target body count 8 000 → 30 000.
